ML Pipeline

How It Works

A 5-step NLP pipeline transforms raw resume text into a ranked list of candidates — the same logic used in production ATS (Applicant Tracking Systems).

1
Text Cleaning & Tokenization
Raw resume text is lowercased, stripped of punctuation, and split into individual word tokens. Mimics spaCy's tokenizer pipeline.
# Tokenization (spaCy / NLTK) text = resume.lower() text = re.sub(r'[^a-z0-9#+.\s]', ' ', text) tokens = text.split() # ["python", "tensorflow", ...]
2
Skill Extraction (Keyword Matching)
Each required skill is matched against the resume using regex with word boundaries. Matched skills are flagged green; missing ones are flagged red as skill gaps.
# Regex skill extraction for skill in required_skills: pattern = rf"\b{re.escape(skill)}\b" if re.search(pattern, resume, re.I): matched.append(skill) # ✓ green else: missing.append(skill) # ✗ red — skill gap
3
TF-IDF Vectorization
The job description and each resume are converted into TF-IDF vectors — numerical representations that weight terms by how unique they are across all documents.
# scikit-learn TF-IDF from sklearn.feature_extraction.text import TfidfVectorizer vectorizer = TfidfVectorizer(stop_words='english') matrix = vectorizer.fit_transform([job_desc] + resumes) jd_vector = matrix[0] # Job description res_vectors = matrix[1:] # All resumes
4
Cosine Similarity
Measures the angle between the JD vector and each resume vector. A score of 1.0 means identical content; 0.0 means no overlap. Scaled to 0–30 points.
# Cosine similarity (scikit-learn) from sklearn.metrics.pairwise import cosine_similarity scores = cosine_similarity(jd_vector, res_vectors) # scores[i] → similarity of resume i with the JD # Range: 0.0 (no match) → 1.0 (perfect match)
5
Weighted Scoring & Ranking
Four signals are combined into a final score out of 100. Candidates are sorted descending — the highest scorer is ranked #1.
# Weighted composite score score = ( (matched / total_skills) * 50 + # Skill match 50% cosine_sim * 30 + # TF-IDF sim 30% experience_fit_bonus + # Experience 10% keyword_density_bonus # Keyword hits 10% ) ranked = sorted(candidates, key=lambda c: c.score, reverse=True)

Scoring Model

How Scores Are Calculated

The final score is a weighted blend of four signals — each targeting a different quality dimension of a candidate's fit.

50%
Skill Match
Ratio of required skills found in the resume. Core signal — if the skills aren't there, nothing else compensates.
30%
TF-IDF Similarity
Cosine similarity between the job description and resume TF-IDF vectors. Captures semantic overlap beyond exact skill keywords.
10%
Experience Fit
Candidate's years of experience matched against the selected level (Junior / Mid / Senior). Over- or under-qualified both reduce this score.
10%
Keyword Density
Proportion of job-title and skill keywords present in the resume token set. Rewards candidates who naturally use industry terminology.
